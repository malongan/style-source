03 / 流程

完整工作流

从灵感到上线，五步完成

S1 收集

灵感 → 风格文件 + 预览图

设计师从推文 / 即梦 / Meigen 收集提示词，生成 .md 风格文件和预览图，存入 style-source/styles/。

prompt-collector · jimeng_extractor.py
S2 策划

需求 → 文案方案

策划师输出 2-3 个差异化方案（主标题、Slogan、画面描述、色彩建议），通过 GitHub Manager 查询 IP 档案。

chat_with_agent → 查 IP 特征 + 生成规则
S3 设计

方案 → 图生图

设计师带上 IP 参考图做图生图，保持角色特征一致性，生成最终图片。

ip-kv-design-v3 · 只写风格 / 动作，不写外貌特征
S4 提交

生图路径 → 提交 GitHub Manager

设计师通过 chat_with_agent 把图片路径发给 GitHub Manager，由我处理 WebP 转换和仓库操作。

chat_with_agent(to_agent="github-manager") · 600s 超时
S5 上线

数据生成 → Gallery 更新 → 上线

运行 generate_data.py + build_gallery.py 自动生成数据，创建 PR，更新 GitHub Pages。

git push · GitHub Pages 自动部署
04 / 协作

Agent 协作模式

Agent 之间如何通信与配合

查询 IP 档案

设计师 / 策划师 → GitHub Manager

chat_with_agent(to_agent="github-manager",
  text="查一下 IP002 莉莉的档案")
返回：角色特征、生成规则、参考图 URL
替换封面 / 发布

设计师 → GitHub Manager

chat_with_agent(to_agent="github-manager",
  text="替换 IP007 大耳狗封面：/path/to/img.png",
  timeout=600)
我转 WebP、更新预览图、Push
策划方案

设计师 → 策划师

chat_with_agent(to_agent="content-creator",
  text="为 IP003 芒果仔策划露营主题 KV...")
返回：2-3 个完整文案方案
更新风格页面

设计师 → GitHub Manager

chat_with_agent(to_agent="github-manager",
  text="新风格已就绪，请更新页面")
运行 generate_data.py → build_gallery.py → PR
风格收集 → 设计师

GitHub Manager → 设计师

chat_with_agent(to_agent="painter",
  text="收集这个提示词：https://x.com/xxx")
设计师提取推文 → 生成风格文件 + 预览图 → 提交回 GitHub Manager
创建 IP → 设计师

GitHub Manager → 设计师

chat_with_agent(to_agent="painter",
  text="请为 IP007 大耳狗生成一个 3D 卡通头像")
设计师生图 → chat_with_agent 提交回 → 我转 WebP → 更新封面
