03 / 流程

完整工作流

从情报收集到交付评分，八步形成创作闭环。每个步骤由对应 Agent 驱动，用户全程参与评判，优质方案评分登记以便复用。

S1

收集风格与 IP

新闻助手抓取热门风格趋势；GitHub Manager 收集新风格、新增 IP 入库。

S2

用户发出设计需求

用户提出设计需求（海报、KV、Banner 等），GitHub Manager 记录并通知设计师。

S3

策划师出文案方案

设计师喊策划师协作，输出 2-3 个差异化方案（主标题、Slogan、画面描述、色彩建议）。

S4

设计师匹配风格

设计师从风格库选择合适的风格，搭配 IP 角色，构造完整 prompt。

S5

提交 Tuzi API 出图

将完整 prompt 提交给 Tuzi API，等待出图完成（超时 600s）。

S6

交付用户评判

将生成的图片发给用户查看，用户反馈满意或修改意见。不满意则回到 S3 优化。

S7

评分登记

用户满意的结果，对 prompt 评分并登记，下次优先使用高分方案。

S8

上线发布

GitHub Manager 处理 WebP 转换，运行 generate_data.py，推送 GitHub Pages。
