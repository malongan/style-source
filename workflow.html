03 / 流程

完整工作流

从灵感到上线，五步完成

S1 收集

灵感 → 风格文件 + 预览图

设计师从推文 / 即梦收集提示词，生成 .md 风格文件和预览图。

prompt-collector
S2 策划

需求 → 文案方案

策划师输出 2-3 个方案，通过 GitHub Manager 查询 IP 档案。

chat_with_agent
S3 设计

方案 → 图生图

设计师带 IP 参考图做图生图，保持角色一致性。

ip-kv-design-v3
S4 提交

生图 → 提交 GitHub Manager

设计师把图片路径发给 GitHub Manager，由我处理 WebP 和仓库操作。

chat_with_agent · 600s 超时
S5 上线

数据生成 → Gallery 更新

运行 generate_data.py + build_gallery.py，创建 PR，更新 Pages。

git push
04 / 协作

Agent 协作模式

Agent 之间如何通信与配合

查询 IP 档案

设计师 / 策划师 → GitHub Manager

chat_with_agent(to_agent="github-manager", text="查 IP002 莉莉的档案")
返回：角色特征、生成规则、参考图 URL
替换封面

设计师 → GitHub Manager

chat_with_agent(to_agent="github-manager",
text="替换 IP007 封面：/path/to/img.png")
转 WebP、更新预览图、Push
策划方案

设计师 → 策划师

chat_with_agent(to_agent="content-creator",
text="为 IP003 策划露营主题 KV")
返回：2-3 个完整文案方案
更新风格页面

设计师 → GitHub Manager

chat_with_agent(to_agent="github-manager", text="新风格已就绪")
generate_data.py → build_gallery.py → PR
风格收集

GitHub Manager → 设计师

chat_with_agent(to_agent="painter",
text="收集提示词：https://x.com/xxx")
提取推文 → 生成风格 + 预览图 → 提交回
创建 IP

GitHub Manager → 设计师

chat_with_agent(to_agent="painter",
text="为 IP007 生成 3D 卡通头像")
生图 → 提交回 → 转 WebP → 更新封面
