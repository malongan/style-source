完整工作流

从灵感到上线，五步完成

S1
收集 · Painter

灵感 → 风格文件 + 预览图

设计师从推文/即梦/Meigen 收集提示词，生成 .md 风格文件和预览图，存入 style-source/styles/。

prompt-collector · jimeng_extractor.py
S2
策划 · Content Creator

需求 → 文案方案

策划师输出 2-3 个方案（标题、Slogan、画面描述、色彩），通过 GitHub Manager 查询 IP 档案。

chat_with_agent → 查 IP 特征 + 生成规则
S3
设计 · Painter

方案 → 图生图

设计师带 IP 参考图做图生图，保持角色特征一致性，生成最终图片。

ip-kv-design-v3 · 只写风格/动作，不写外貌
S4
提交 · Painter → GitHub Manager

生图路径 → 提交给我

设计师通过 chat_with_agent 把图片路径发给我，我处理 WebP 转换和后续操作。

chat_with_agent(to_agent="github-manager") · 600s 超时
S5
上线 · GitHub Manager

数据生成 → Gallery 更新 → 上线

运行 generate_data.py + build_gallery.py，创建 PR，更新 GitHub Pages。

git push · GitHub Pages 自动部署

Agent 之间如何通信与配合

设计师 / 策划师 → GitHub Manager

chat_with_agent(to_agent="github-manager", text="查一下 IP002 莉莉的档案")
返回：角色特征、生成规则、参考图 URL
替换封面 / 发布

设计师 → GitHub Manager

chat_with_agent(to_agent="github-manager", text="替换 IP007 大耳狗封面：/path/to/img.png", timeout=600)
我转 WebP、更新预览图、Push
策划方案

设计师 → 策划师

chat_with_agent(to_agent="content-creator", text="为 IP003 芒果仔策划露营主题 KV...")
返回：2-3 个完整文案方案

设计师 → GitHub Manager
