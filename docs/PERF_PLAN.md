# 🚀 Gallery 性能与体验优化计划

> 基于 2026-07-02 全面审计结果

---

## 一、现状指标

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 首屏资源 | 143KB（HTML 88KB + JS 32KB + CSS 23KB） | < 50KB |
| 图片加载 | 168 张全部加载，无懒加载 | 可见区域加载 |
| JS 加载 | 同步阻塞渲染 | 异步/defer |
| 内联数据 | 73KB FALLBACK_DATA 写入 HTML | 移除，按需 fetch |
| 图片 | 168 全尺寸 WebP (14.6MB) + 168 缩略图 (2.8MB) | - |
| 平均 WebP | 90KB 全尺寸 / 17KB 缩略图 | 60KB / 12KB |
| 瀑布流 | 全量渲染 168 张 | 按需渲染 |
| CDN | GitHub Pages（无专用 CDN） | 建议 Cloudflare/jsDelivr |

---

## 二、优化方向

### 优先级 P0 🔴 — 必须做，高影响低风险

#### 1. ✅ 图片懒加载
- **问题**: 页面加载全部 168 张图片，卡顿
- **方案**: `loading="lazy"` 属性注入 + `IntersectionObserver`
- **影响**: 首屏加载从 14.6MB → 约 500KB（首屏 5-6 张）
- **工作量**: 小，改 `gallery-runtime.js` 中图片模板即可
- **风险**: 极低，纯前端优化

#### 2. ✅ 移除内联 FALLBACK_DATA
- **问题**: HTML 中内联 73KB JSON 数据，阻塞渲染且不可缓存
- **方案**: 
  - 完全移除 HTML 中的 `__FALLBACK_DATA__`
  - 改为 `fetch('data/styles.json')` 异步加载
  - 加骨架屏兜底
- **影响**: HTML 从 88KB → 15KB，减少 83%
- **工作量**: 中，需改 `build_gallery.py` + `gallery.js`
- **风险**: 低，JS 已有异步加载逻辑

#### 3. ✅ JS/CSS 异步加载
- **问题**: `gallery-runtime.js` 同步 `<script>` 加载，阻塞渲染
- **方案**: 改为 `<script defer>` + CSS font-display: swap
- **影响**: 首屏渲染时间减少 200-400ms
- **工作量**: 小，改 `build_gallery.py` 模板
- **风险**: 极低

#### 4. ✅ HTML 压缩
- **问题**: gallery.html 含大量空白字符和注释
- **方案**: `build_gallery.py` 输出时压缩 HTML
- **影响**: 15KB → 约 10KB
- **工作量**: 小，加几行 Python
- **风险**: 极低

---

### 优先级 P1 🟡 — 建议做，中等影响低风险

#### 5. URL 参数过滤
- **问题**: 无法通过 URL 分享筛选/搜索状态
- **方案**: 
  - 监听 filter/search/sort 变化 → 更新 `URLSearchParams`
  - 页面加载时读取 URL 参数自动应用
  - 支持：`?category=painting&q=3d&sort=newest`
- **影响**: 可分享搜索结果，提升协作体验
- **工作量**: 中
- **风险**: 低

#### 6. IntersectionObserver 优化
- **问题**: 可能存在基于 scroll 事件的手动实现
- **方案**: 替换 scroll 事件为 IntersectionObserver
- **影响**: 减少主线程压力，帧率提升
- **工作量**: 中，需审查现有 scroll 监听
- **风险**: 低

#### 7. WebP 回退（`<picture>` 标签）
- **问题**: 部分老旧浏览器不支持 WebP
- **方案**: `<picture> <source srcset="img.webp" type="image/webp"> <img src="img.jpg" ...> </picture>`
- **影响**: 兼容性提升，但增加 HTML 体积
- **工作量**: 大，需改 `build_gallery.py` 卡片模板
- **风险**: 低

#### 8. 键盘导航增强
- **问题**: 除 Lightbox 左右键外，无键盘导航
- **方案**: Tab/Enter/Escape/ / 快捷键
- **影响**: 无障碍体验提升
- **工作量**: 中
- **风险**: 低

---

### 优先级 P2 🔵 — 值得做，但与当前架构关联较大

#### 9. 无限滚动/分页
- **问题**: 168 张卡片全量渲染，DOM 节点过多
- **方案**: 首次渲染 20-30 张，IntersectionObserver 追加
- **影响**: 首屏 DOM 节点从 168 → 30
- **工作量**: 大，涉及与搜索/分类/排序/收藏联动
- **风险**: 中

#### 10. 渐进式图片加载
- **问题**: 图片瞬间显示，无过渡
- **方案**: 先展示缩略图占位，再加载全尺寸，模糊过渡
- **影响**: 感知加载速度提升
- **工作量**: 中
- **风险**: 低

#### 11. CDN 加速
- **问题**: GitHub Pages 全球访问延迟高
- **方案**: Cloudflare R2 / jsDelivr / Cloudflare Pages
- **影响**: 全球加载速度提升 50-80%
- **工作量**: 大，部署流程变更
- **风险**: 中

---

### 优先级 P3 ⚪ — 锦上添花

| 项目 | 说明 | 工作量 |
|------|------|--------|
| PWA / Service Worker | 离线缓存，App 安装 | 大 |
| 分享功能 | 分享卡片到社交媒体 | 中 |
| 统计/分析 | Plausible / Umami 自托管 | 小 |
| 一键复制提示词 | 点卡片复制 prompt | 小 |
| 图片放大镜 | 鼠标悬停局部放大 | 中 |
| 对比功能 | 同时选择多个风格对比 | 大 |
| 随机发现 | 随机展示一个风格 | 小 |
| 打印友好 | 打印优化 CSS | 小 |

## 三、预期效果

| 指标 | 优化前 | P0 后 | P0+P1 后 | 全部 |
|------|--------|-------|----------|------|
| 首屏 HTML | 88KB | 15KB | 15KB | 12KB |
| 首屏资源 | 143KB | 70KB | 70KB | 50KB |
| 首屏图片加载 | 14.6MB | ~500KB | ~500KB | ~200KB |
| 首屏渲染 | ~2s* | ~0.8s* | ~0.8s* | ~0.5s* |
| Lighthouse 性能 | ~50* | ~70* | ~80* | ~90* |

> \* 估算值，以实际测试为准

---

## 四、不推荐的方案

| 方案 | 原因 |
|------|------|
| Vue/React 重写 | 当前纯 JS IIFE 架构已稳定，重写成本高收益低 |
| SSR/SSG 生成 | 静态页面已预生成，168 页面增加复杂度 |
| 图片 CDN 迁移 | 需域名/部署变更，P3 优先级 |
| 数据库后端 | 纯静态方案已满足需求 |
